🎯 功能概述

組隊比賽功能是本系統的核心模組，提供完整的多隊伍競技管理解決方案。與一般挑戰賽不同，組隊比賽具備以下特色：

  • 多隊參與：支援多支隊伍同時參與競賽
  • 正式賽制：採用循環賽或淘汰賽制度（淘汰賽施工中）
  • 完整管理：從報名到結果分析的全流程管理
  • 靈活模式：支援集中式與分散式兩種比賽模式
  • 自動化：對戰表自動產生、桌次智能分配

🏅 比賽類型差異

⚠️ 重要特色：本系統支援兩種獨特的比賽進行模式，這是與其他桌球管理系統的主要差異。

🎪 集中式比賽

定義：所有參賽隊伍在同一時間、同一地點進行比賽

  • 時間安排：統一比賽日期與時段
  • 場地需求：需要足夠的球桌同時進行多場比賽
  • 桌次分配：系統自動分配桌次，避免衝突
  • 現場管理：主辦方需要現場協調與監督
  • 比賽節奏：快速完成所有對戰

🗓️ 分散式比賽

定義：對戰雙方自行約定時間完成比賽

  • 時間彈性：參賽者可自由安排比賽時間
  • 場地彈性：不限定特定場地進行
  • 桌次管理：不需考慮桌次衝突問題
  • 比賽週期：比賽時間可延長至數週或數月
  • 約戰機制：透過系統約戰功能協調比賽時間
比較項目 集中式比賽 分散式比賽
適用場合 正式錦標賽、團體活動 長期聯賽、跨地區比賽
時間控制 嚴格統一 彈性自由
場地需求 集中場地、多球桌 分散場地皆可
管理複雜度 現場管理集中 過程管理分散
完成速度 一天內完成 數週至數月
💡 選擇建議：
  • 選擇集中式：參賽者地理位置相近、希望快速完成比賽、有足夠場地資源
  • 選擇分散式：參賽者時間難以統一、跨地區參與、希望延長比賽樂趣

⚙️ 建立比賽

⚠️ 權限要求：只有系統管理員或球場管理者可以建立比賽。
💡 重要說明：集中式與分散式比賽的操作方式完全相同，差異僅在於比賽進行的時間安排：
  • 集中式：所有隊伍統一時間到場比賽
  • 分散式：對戰雙方自行約定時間進行比賽
兩種模式使用相同的系統功能和操作流程。

建立比賽步驟

1基本資訊設定

  • 比賽名稱：建議包含日期和性質，如「2025年春季循環賽」
  • 球場名稱：系統根據建立者身份自動設定
  • 比賽規則：詳細說明比賽規則、積分方式等
  • 報名結束日：設定隊伍報名的截止日期

2參賽規模設定

  • 預計隊數：預估參賽隊伍數量
  • 每隊人數：每支隊伍的參賽人數

3賽制與場地設定

  • 賽制類型：目前主要支援循環賽制
  • 比賽球桌數：影響集中式比賽的桌次分配
  • 總點數：每場對戰的局數設定

4每點賽制詳細設定

✅ 建立成功後：系統會自動跳轉到賽程控制區，比賽狀態設為「招募中」。

👥 隊伍管理

進入參賽區

🏠 參賽入口：參賽者需從首頁進入「參賽區」開始報名流程。

隊伍報名流程

參賽者進入比賽 → 建立隊伍 → 邀請隊員 → 隊員回應 → 隊長確認人員到位

1參加隊伍或建立隊伍

  • 參加現有隊伍：可選擇加入已建立的隊伍（如有空缺）
  • 建立新隊伍：在隊伍數未達上限時可建立新隊伍
  • 每位參賽者只能在同一比賽中參加一支隊伍
  • 建立者自動成為隊長，擁有隊伍管理權限

2隊長邀請隊員

  • 隊長可邀請同球場的其他成員
  • 邀請功能可在隊伍管理頁面中使用
  • 系統會自動篩選可邀請的成員（排除已參賽或已被邀請者）

3隊員接收邀請

📱 邀請通知方式：

  • 被邀請者在比賽頁面會看到邀請資訊
  • 個人資訊頁面會顯示邀請提醒
  • 被邀請者可選擇「接受邀約」或「不克參加」
  • 回應後系統會即時更新隊伍狀態
  • 拒絕後仍可被重新邀請

4隊長管理功能

🔧 隊長專用功能：

  • 查看成員狀態：即時了解邀請回應情況
  • 移除成員：可移除已加入的隊員
  • 最終確認：人數足夠時確認隊伍完成

5確認人員到位

⚠️ 關鍵步驟：當隊伍達到規定人數時，隊長會收到通知，必須進行最終確認。
✅ 確認完成：隊長點擊「人員已到位」後，該隊伍狀態變更為「已確認」，可參與對戰表產生。

隊伍狀態管理

  • 招募中：隊伍正在招募成員
  • 已確認：隊長已確認人員到位
  • 可開始：符合產生對戰表的條件

📋 對戰表產生

⚠️ 重要時機：只有當所有隊伍都確認「人員已到位」時，管理員才能產生對戰表。產生後將無法更改隊伍名單。

產生條件檢查

1一般使用者確認狀態

📊 如何知道各隊準備狀況：參賽者可在比賽頁面查看所有隊伍的準備狀態。
✅ 準備完成標示：當所有隊長都確認人員到齊後，頁面會顯示「等待對戰表」狀態。

2管理員產生對戰表

🔧 管理員操作路徑：

  • 方式一：從首頁 → 管理員專區 → 賽程控制區
  • 方式二：從個人資訊頁面的通知連結

3對戰表產生過程

🔄 循環賽對戰表產生邏輯：
  • 每支隊伍都會與其他所有隊伍對戰一次
  • 系統智能分配輪次，盡量平衡每輪比賽數量
  • 考慮桌次限制，最佳化比賽排程
  • 對戰表考慮平衡每隊出賽時間

4比賽詳情建立

  • 根據設定的總點數建立比賽詳情
  • 每點比賽標註單打或雙打類型
  • 初始化桌次分配系統
✅ 產生成功後：比賽狀態變更為「人員安排中」，進入名單編排階段。

📝 名單安排

📋 名單安排階段：對戰表產生後，各隊隊長需要為每場比賽安排具體的出賽人員。

名單編排流程

1查看名單狀況

📋 名單狀況查看：在比賽戰況室中選擇「名單狀況」可查看最新的對戰安排。

2編排出賽名單

🔧 隊長編輯入口：

  • 方式一：從名單狀況頁面點擊「編輯名單」
  • 方式二：從個人資訊頁面的提醒連結

3名單安排要點

  • 單打項目：每項目選擇1位出賽選手
  • 雙打項目：每項目選擇2位出賽選手
  • 重複使用：同一選手可在多個項目出賽
  • 視覺提示：已在其他項目選中的選手會標示「已選」
  • 彈性安排：可不足額選擇，但系統會警告
⚠️ 重要：名單安排與桌次分配
只有當對戰雙方都完成名單安排時，該場比賽才會被系統分配桌次。這是自動桌次分配機制的關鍵。

戰況室功能說明

  • 搜尋功能：可按隊伍名稱或成員姓名搜尋特定比賽
  • 桌次顯示：顯示目前分配的桌次或「Next」(待分配)

桌次管理系統

🎯 智能桌次分配：系統根據可用球桌數量自動分配比賽桌次

  • 優先分配：雙方都已提交名單的比賽優先分配桌次
  • 動態調整：比賽結束後，空出的桌次自動分配給下一場
  • Next標示：標示為「Next」的比賽將在有桌次空出時優先安排

約戰功能整合

⏰ 約戰功能：特別適用於分散式比賽，讓對戰雙方可以協調比賽時間。
📖 查看約戰功能詳細說明

約戰使用時機

  • 分散式比賽：雙方需要協調具體比賽時間
  • 集中式比賽：可用於賽前溝通或調整
  • 特殊情況：需要延期或重新安排的比賽

比賽進行操作

1前往比賽

  • 點擊「前往比賽」按鈕進入計分頁面
  • 系統自動判斷單打或雙打模式
  • 參數自動帶入，包含選手資訊

2計分與記錄

  • 使用系統內建計分功能
  • 比分自動儲存並同步到戰況室
  • 完成後自動更新排名統計
✅ 權限控制：
  • 管理員：可操作任何比賽
  • 參賽者：可操作任何場次（互相擔任裁判）
  • 非參賽者：僅能觀看，無法操作

📊 比分管理與結果分析

比分表功能

📊 即時比分統計：系統提供完整的比分統計與排名分析功能。
📖 查看比分表功能詳細說明

比分表特色

  • 即時更新：比賽結束後立即反映在比分表中
  • 多層排名：依勝場數、直接對戰、勝局數進行排名
  • 視覺提示：不同顏色標示比賽狀態與結果
  • 進度追蹤：粉紅色背景標示未完成的比賽

管理員比分編輯

⚠️ 管理員專用：比分編輯功能僅供管理員使用，用於修正錯誤或特殊情況處理。

比賽結束管理

1結束條件檢查

  • 所有對戰都已完成並記錄比分
  • 沒有未完成的比賽（無粉紅色背景格子）
  • 管理員確認所有結果無誤

2確認比賽結束

✅ 管理員操作：點擊「結束比賽」按鈕，系統狀態變更為「比賽已結束」

3詳細對戰記錄

📝 比賽結束後解鎖：系統提供詳細的個人對戰記錄，包含每局選手配對和結果。

🔄 完整操作流程

集中式比賽完整流程

管理員建立比賽 → 設定比賽參數 → 開放報名 ↓ 隊長建立隊伍 → 邀請隊員 → 隊員回應 → 確認人員到位 ↓ 管理員產生對戰表 → 隊長編排名單 → 系統分配桌次 ↓ 比賽當日集中進行 → 即時計分記錄 → 比賽結束 ↓ 管理員確認結束 → 查看詳細記錄 → 比賽歷史歸檔

分散式比賽完整流程

管理員建立比賽 → 設定比賽參數 → 開放報名 ↓ 隊長建立隊伍 → 邀請隊員 → 隊員回應 → 確認人員到位 ↓ 管理員產生對戰表 → 隊長編排名單 ↓ 對戰雙方約定時間 → 分別進行比賽 → 記錄比分 ↓ 所有對戰完成 → 管理員確認結束 → 查看最終結果

關鍵時間節點

  • 報名截止：隊伍建立的最後期限
  • 名單確認：隊長確認人員到位的時機
  • 對戰表產生：不可逆的重要節點
  • 名單編排：影響桌次分配的關鍵步驟
  • 比賽結束：開放詳細記錄查看

❓ 常見問題

比賽建立相關

Q：為什麼我無法建立比賽？

A：只有系統管理員或球場管理者才能建立比賽。請確認您的帳號權限。

Q：比賽建立後可以修改設定嗎？

A：在產生對戰表前，管理員可以修改大部分設定。對戰表產生後，基本設定將無法更改。

隊伍管理相關

Q：可以同時參加多支隊伍嗎？

A：在同一場比賽中，每個人只能參加一支隊伍。但可以同時參加不同比賽的隊伍。

Q：隊長可以更換嗎？

A：目前系統不支援隊長轉移功能。如需更換，請聯繫系統管理員處理。

Q：隊員拒絕邀請後還能重新邀請嗎？

A：可以。隊長可以重新邀請已拒絕的隊員。

名單編排相關

Q：為什麼我的比賽沒有分配到桌次？

A：只有對戰雙方都完成名單編排時，系統才會分配桌次。請確認對手是否已提交名單。

Q：名單編排後可以修改嗎？

A：在比賽開始前，隊長可以重新編輯名單。比賽開始後將無法修改。

比賽進行相關

Q：非參賽者可以幫忙計分嗎？

A：為了公平性，只有參賽者和管理員可以進行計分操作。

Q：比分記錄錯誤怎麼辦？

A：請聯繫管理員使用比分編輯功能進行修正。

集中式 vs 分散式比賽

Q：如何選擇比賽模式？

A：系統本身不區分模式，差異在於執行方式：

  • 集中式：所有人同一時間到場比賽
  • 分散式：對戰雙方自行約定時間比賽

Q：分散式比賽需要約戰嗎？

A：不是必須的。對戰雙方可以透過其他方式聯繫，約戰功能只是提供方便的協調工具。

技術問題

Q：資料沒有即時更新怎麼辦？

A：請嘗試重新整理頁面。如果問題持續存在，請聯繫技術支援。

Q：忘記密碼怎麼辦？
